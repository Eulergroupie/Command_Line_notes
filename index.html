Codecademy Learn the Command Line

The command line is a text interface for the computer's operating system. To access the command line, we use the terminal. From the command line, you can navigate through files and directories (folders) on your computer.
A filesystem organizes a computer's files and directories into a tree structure. It starts with the root directory. Each parent directory can contain more child directories and files.

Navigating the File System

We will focus on Unix-based systems.
Five common commands:

  • pwd "Print Working Directory" outputs the name of the current working directory
  • ls lists all files and directories in the working directory
    • option -a shows all files and directories including hidden ones (start with a dot)
    • option -l shows a long list with 7 columns of information
      • access rights: rules that govern read, write, and execute
      • hard links: number of child directories and files
      • owner's username
      • owner's group
      • file size
      • date & time modified
      • file or directory name
    • option -t shows a list sorted by the time and date last modified
    • option -alt shows all of the above at once
  • cd pathname switches you into the directory you specify
    • . is a special directory name that means "this directory"
    • .. is a special directory name that means "one directory up"
    • ~ is a special directory name that means "home" (i.e. where your stuff begins)
    • / is a special directory name that means "root" (i.e. the top of the file system)
  • mkdir directory_name creates a new directory in the working directory
  • touch filename creates a new file inside the working directory
When a file, directory or program is passed into a command, it is called an argument.
Options are dashed letters that follow a command and modify its behavior.

Viewing and Changing the File System

We can also use the command line to copy, move, and remove files and directories.

  • cp source(s) destination copies files
  • mv source(s) destination moves and renames files
  • rm filename removes files
    • option -r recursively removes directories that are not empty
    • option -f forces the remove without prompting if read-only

Often wildcards like * (meaning any number of any characters) can be used in place of, or combined with filenames

Redirecting Input and Output

Through redirection you can direct the input and output of a command to and from other files and programs, and chain commands together in a pipeline.

Typing echo "Hello" accepts the string Hello as standard input and redirects it to standard output. Here is some information about standard streams:

  1. Standard input or stdin is information inputted into the terminal through the keyboard or input device
  2. Standard output or stdout is the information outputted after a process is run
  3. Standard error or stderr is an error message outputted by a failed process
Redirection reroutes standard input, standard output, and standard error to or from a different location.

Some more useful commands for manipulating text:

  • Typing cat filename outputs the contents of a file to the terminal (stdout).
  • Typing wc filename will return the number of newlines (lines), words, and bytes (characters) in a file.
  • Typing sort filename orders a file alphabetically for the standard output.
  • Typing uniq filename filters out adjacent, duplicate lines in a file.
  • Typing head filename outputs the first 10 lines of a file.
  • Typing grep pattern filename searches files for lines that match a pattern and returns the results. The pattern may be a basic regular expression. e.g. grep Mount mountains.txt
    • Option -i enables the command to ignore case.
    • Option -R enables the command to recursively search directories, returning filenames and the matching text.
    • Option -l enables the command to return the first filename with a match. Combined with -R, multiple filenames may be returned.
  • Typing sed pattern_string filename calls the stream editor to search a file and replace word(s) found according to the pattern_string. e.g. sed 's/snow/rain/' forests.txt would replace the first instance of snow in the file forests.txt with the word rain. 's' stands for substitution. When 'g' (for global) is added to the end of the pattern, all instances are replaced. e.g. sed 's/snow/rain/g' forests.txt

The following are the standard redirect commands:

  • The > command redirects the standard output to a file, either creating it if it doesn't exist or writing over it.
  • The >> command redirects the standard output to append a file.
  • The < command redirects output from a command on the right to be input for the command on the left.
  • The | is called a pipe and it redirects the output from a command on the left and pipes it in as input to the command on the right. Piped commands can also be chained together. e.g. cat volcanoes.txt | wc | cat > islands.txt
